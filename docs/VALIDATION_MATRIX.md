# Validation Matrix – MontageZeit

Prüfmatrix **pro Änderungstyp**: welche Checks minimal, erweitert und
manuell laufen müssen. Ergänzt die testbereich-orientierte Übersicht in
[`docs/CODEMAP.md`](CODEMAP.md) (dort: „welche Tests gibt es"; hier: „welche
Checks muss ich für diese Änderung laufen lassen").

Grundregel aus [`AGENTS.md`](../AGENTS.md): kleinste sinnvolle Prüfung zuerst,
vor Handoff/PR auf den vollen lokalen Quality Gate eskalieren:

```bash
bash scripts/hooks/run_local_quality_gate.sh   # detekt + lint + testDebugUnitTest + assembleDebug
```

## Vor jedem Handoff / PR

```bash
./gradlew detekt
./gradlew lint
./gradlew :app:testDebugUnitTest
./gradlew assembleDebug
bash scripts/hooks/run_local_quality_gate.sh
```

Ergebnis (grün/rot/nicht gelaufen + Grund) im PR-Body und in
[`docs/AGENT_HANDOFF_TEMPLATE.md`](AGENT_HANDOFF_TEMPLATE.md) festhalten —
nie stillschweigend als „gelaufen" behaupten, wenn ein Check nicht ausgeführt
werden konnte.
